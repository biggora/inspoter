Необходимо разработать (PRD, dev plan, documentation etc.) и создать специализированную панель управления.
Панель управления должна содержать следующие разделы:

- Bookmarks — каталог ссылок на ресурсы. каталог должен иметь категоризацию. должна быть возможность указать иконку, наименование и описание для каждой ссылки. По такому же принципу как Dashy, Homarr etc..
- Domains — панель мониторинга и управления доменными именами. должна быть возможность управлять доменными именами, DNS записями (Cloudflare, Hetzner, Godaddy).
- Servers — панель мониторинга и управления VPS серверами. должна быть возможность управлять серверами, их конфигурациями и состоянием (Hetzner).
- Mail — панель мониторинга и управления почтой. должна быть возможность просматривать почту, фильтровать и сортировать её по различным критериям. должен поддерживается webhook API, для отправки сообщений из сторонних систем.
- Messages — панель мониторинга и управления сообщениями. С таким же принципом как у Discord, иметь такую же структуру — категории и каналы. должен поддерживается webhook API, для отправки сообщений из сторонних систем.
- Logs — панель мониторинга и управления логами. должна быть возможность просматривать логи, фильтровать и сортировать их по различным критериям. должен поддерживается webhook API, для отправки сообщений из сторонних систем.
- Alerts — панель мониторинга и управления уведомлениями. должен иметь категоризацию. должна быть возможность просматривать уведомления, фильтровать и сортировать их по различным критериям. должен поддерживается webhook API, для отправки alerts из сторонних систем.
- Workspaces — рабочие пространства для организации ресурсов. многие-ко-многим пользователи↔пространства. invite-only добавление пользователей.
